## Intro

Python tools for EDH/Commander deckbuilders that aggregate data from EDHRec. Enter a commander name and the tool fetches decklists, counts card frequency across those decks, and sorts cards by type — producing text files that can be imported into Moxfield, Archidekt, MPC Autofill, etc.

The primary interface is a Streamlit web UI. Legacy CLI scripts are also available for local use.

## Web UI

The web app is live at **<https://magic.spicyfajitas.com>**

Features:

- Enter a commander name, number of recent decks to analyze, and budget range
- Fetches and aggregates EDHRec decklists automatically
- Dashboard with card frequency charts
- Visual card browser with hover previews (Scryfall images)
- Download individual output files or all as a ZIP

## Deployment

### Docker Compose (recommended)

The `docker-compose.yaml` in this repo runs the app plus an auto-updater.

```bash
docker compose up -d
```

Services started:

| Service | Purpose | Port |
| ------- | ------- | ---- |
| `edhrec-analyzer` | Streamlit web app | 8501 |
| `watchtower` | Auto-pulls updated images hourly | — |

Mount your local `cache/` and `output/` directories to preserve them across container restarts.

### Docker run (standalone)

```bash
docker pull ghcr.io/spicyfajitas/edhrec-deck-analyzer:latest
docker run -d --name edhrec-analyzer -p 8501:8501 \
  -v ./cache:/app/cache \
  -v ./output:/app/output \
  ghcr.io/spicyfajitas/edhrec-deck-analyzer:latest
```

## Outputs

The `output/` folder stores text files generated by the tool. Files are grouped by commander name. Each run replaces the previous run's output for the same commander.

### EDHRec Decklists Statistics

From the web UI or `edhrec_decklists_json_to_txt.py` script — aggregates actual decklists from EDHRec and counts how often each card appears across those decks.

![EDHRec Commander Decks List](./files/edhrec-commander-decks-list.png)

![EDHRec Deck by DeckID](./files/edhrec-commander-deck-by-deckid.png)

```text
output/
└── mr-orfeo-the-boulder
    └── edhrec-decklists
        ├── cards_artifact.txt
        ├── cards_creature.txt
        ├── cards_enchantment.txt
        ├── cards_instant.txt
        ├── cards_land.txt
        ├── cards_planeswalker.txt
        ├── cards_sorcery.txt
        ├── commander.txt
        ├── master_card_counts.txt
        └── mr-orfeo-the-boulder-decklists.txt
```

`master_card_counts.txt` lists every card and how many of the analyzed decks included it. `commander.txt` records the run parameters (commander, deck count, price range, timestamp).

## Cache

Deck JSON and Scryfall card type data are cached locally to avoid redundant API calls.

### Cache directory structure

```text
cache/
├── deck_cache/
│   ├── 0d12VwcQATaT8syoJ5MbyA.json
│   └── ...
└── scryfall_cache.json
```

### Deck cache file structure

```json
[
  "1 Mr. Orfeo, the Boulder",
  "1 Abnormal Endurance",
  "1 Aloe Alchemist",
  ...
]
```

### Scryfall data cache structure

```json
{
  "Mr. Orfeo, the Boulder": "Legendary Creature — Rhino Warrior",
  "Arcane Signet": "Artifact",
  "Ash Barrens": "Land",
  ...
}
```

## CLI Scripts (legacy)

The original command-line scripts are still available for local use without the web UI.

### Setup

```bash
python3 -m venv venv-magic-scripts
source venv-magic-scripts/bin/activate
pip install requests tqdm
```

### Usage - EDHRec Recommendations

```bash
python3 edhrec_json_to_txt.py
```

### Usage - EDHRec Decklist Statistics

Edit `commander.txt` with the commander's full card name (e.g. `Mr. Orfeo, the Boulder`), then run:

```bash
# Interactive prompts
python3 edhrec_decklists_json_to_txt.py

# CLI arguments
python3 edhrec_decklists_json_to_txt.py --commander "Mr. Orfeo, the Boulder" --recent 5 --min-price 1 --max-price 100
```

```bash
# Deactivate venv when done
deactivate
```
